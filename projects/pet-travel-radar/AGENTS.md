# 펫 트래블 레이더 데모

## 프로젝트 목표

이 Expo 앱은 한국관광공사 오픈 API 활용 공모전 팀회의를 위한 사전 데모입니다.
핵심 아이디어는 포켓몬 고처럼 사용자의 실제 이동과 주변 반경을 앱 화면에 보여주고, 반려동물과 함께 갈 수 있는 관광지, 산책로, 카페, 식당, 숙소가 반경 안에 들어오면 추천해주는 위치 기반 여행 앱입니다.

## 핵심 기획

- 한국관광공사의 반려동물 동반 여행 데이터를 단순 목록이 아니라 실시간 탐색 경험으로 바꿉니다.
- 반려동물 보호자가 여행 중 바로 갈 수 있는 장소를 거리, 방향, 동반 조건 중심으로 빠르게 확인하게 합니다.
- React Native와 Expo를 사용해 실제 모바일 앱처럼 보여주고, Expo Go로 팀원들이 쉽게 테스트할 수 있게 합니다.

## 현재 데모 범위

- 실제 API 키 없이 목데이터만 사용합니다.
- 실제 지도 SDK 대신 레이더형 지도 화면으로 앱 콘셉트를 표현합니다.
- 내 위치, 탐색 반경, 주변 반려동물 동반 가능 장소, 추천 카드를 표시합니다.
- "앞으로 이동" 버튼으로 사용자의 이동을 시뮬레이션합니다.
- 장소 리스트를 누르면 하단 상세 패널에서 장소 정보와 반려동물 동반 조건을 확인할 수 있습니다.
- Concept 탭에서는 목데이터가 나중에 한국관광공사 API 데이터로 어떻게 교체될 수 있는지 설명합니다.

## 데이터 모델

목데이터는 `data/petTravelPlaces.ts`에 있습니다.

각 장소 데이터는 현재 아래 필드를 가집니다.

- `name`: 장소명
- `category`: Cafe, Walk, Stay, Tour, Food 중 하나
- `lat`, `lng`: 추후 실제 지도와 API 연동에 사용할 좌표
- `distanceM`: 사용자와의 시뮬레이션 거리
- `bearingDeg`: 사용자 기준 시뮬레이션 방향
- `petPolicy`: 반려동물 동반 조건
- `address`: 표시용 주소
- `openNow`: 운영 상태
- `tags`: 추천 태그
- `tip`: 추천 이유 또는 이용 팁

## 구현 원칙

- 팀회의용 데모는 안정성이 중요하므로 실시간 GPS보다 목데이터와 고정된 이동 시뮬레이션을 우선합니다.
- Expo Go에서 바로 실행할 수 있도록 네이티브 의존성 추가는 신중하게 결정합니다.
- 실제 위치 기능을 붙일 때는 먼저 Expo 호환 API를 사용합니다.
- 한국관광공사 API로 교체할 때 화면 컴포넌트는 최대한 유지하고 데이터 소스만 바꾸는 구조를 지향합니다.

## 다음 구현 단계

1. 한국관광공사에서 사용할 반려동물 동반 여행 API 엔드포인트를 확정합니다.
2. API 응답 필드를 현재 `PetTravelPlace` 타입에 매핑합니다.
3. Expo Location으로 실제 GPS 권한과 현재 위치 추적을 추가합니다.
4. 레이더 화면을 실제 지도 모드 또는 AR 카메라 모드로 확장합니다.
5. 장소 상세 화면에 이미지, 전화번호, 운영시간, 동반 조건, 데이터 출처를 추가합니다.
6. 저장한 장소, 추천 코스, 팀원 공유 기능으로 확장합니다.

## 발표용 설명

"이 프로토타입은 실제 API 연동 전 사용자 경험을 먼저 보여주는 데모입니다. 사용자가 도시를 이동하면 앱이 주변 반경을 스캔하고, 한국관광공사 반려동물 동반 여행 데이터에 있는 장소가 가까워졌을 때 추천 카드와 상세 정보를 보여줍니다. 공모전에서의 차별점은 관광 데이터를 단순 검색 목록으로 보여주는 것이 아니라, 반려동물 보호자를 위한 실시간 여행 가이드 경험으로 바꾼다는 점입니다."
